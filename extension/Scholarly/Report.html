1. Abstract

WeDI is a Chrome extension whose purpose is to detect and extract schemas requested by the user. These schemas come written in the RDFa, Microdata and JSON-LD HTML5 language extensions. As such, this extension will detect all these formats.

2. Project Progress

Initially, the project started slow, given that we misunderstood the requirements. We started on the wrong foot, but as we spoke with the professor, things became clearer and the project progressed fast. We split the tasks equally and made sure we coded in the same style, to avoid creating unnecessary confusion when integrating each part of the project.

3. Project Architecture

The project was written using VanillaJS, HTML5 and CSS3. Each functionality is in its own .js file, for modularity. The extension injects these scripts in the current tab, to access the DOM we wish to process. Upon finding a schema, the function transforms it into JSON-LD, and added to a list of JSON's which is returned by the tab to the extension. Lastly, the extension displays each JSON.

4. Testing

The extension was tested on various sites, as well as self made pages, to make sure the functions worked correctly and outputted the expected results. Some of the tested pages are:

  1. Corido
  2. Fika
  3. Schema.org
These pages contains either JSON-LD, RDFa, Microdata, or combinations of them.
