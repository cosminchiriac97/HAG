1. Abstract

WeDI is a Chrome extension whose purpose is to detect and extract schemas requested by the user. These schemas come written in the RDFa, Microdata and JSON-LD HTML5 language extensions. As such, this extension will detect all these formats.

2. Project Progress

Initially, the project started slow, given that we misunderstood the requirements. We started on the wrong foot, but as we spoke with the professor, things became clearer and the project progressed fast. We split the tasks equally and made sure we coded in the same style, to avoid creating unnecessary confusion when integrating each part of the project.

3. Project Architecture

The project was written using VanillaJS, HTML5 and CSS3. Each functionality is in its own .js file, for modularity. The extension injects these scripts in the current tab, to access the DOM we wish to process. Upon finding a schema, the function transforms it into JSON-LD, and added to a list of JSON's which is returned by the tab to the extension. Lastly, the extension displays each JSON in tables, each depth being highlighted with a different color (a legend of that is shown in the settings tab).

3.1 Splitting the project

We split the project in 3 main parts: parsing the DOM for RDFa and transforming it into JSON-LD, parsing the DOM for Microdata and transforming it into JSON-LD, and the front-end part which included parsing JSON-LD's.

3.2 RDFa

To parse the DOM for RDFa, we used a recursive algorithm which searches for the elements annotated with "typeof". The element is then checked for properties, and then processed accordingly. The algorithm then goes further to the processed element's children.

3.3 JSON-LD

For the JSON-LD parsing we use quite an easy approach. We simply extract the script tagged elements, we filter them by their type ("application/ld+json"), and then we just parse the inner JSONs.

3.4 Microdata

The extract method is very similar to the RDFa one. The algorithm iterates thru the DOM in search for the specified types, once he founds it, then it recursively collects data further in order to create ultimately, a json. All the JSON-LDs generated are sent to the extention's script that transforms it into a table in the user interface.

3.5 Exports

All the resulted data after scrapping a webpage accordingly to the user's filters can be exported as either one of the formats ( Microdata, JSON-LD and RDFa). For both Microdata and RDFa there is a script for each one that receives a JSON-LD and parses it into a HTML file, or straight JSON.

  2. Fika

  4. Our very own html testing page

These pages contains either JSON-LD, RDFa, Microdata, or combinations of them. In order to compare results, we used Google's Testing Tool, as well as extensions featuring the same functionalities: OpenLink Structured Data Sniffer Structured Data Testing
